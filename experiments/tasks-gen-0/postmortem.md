## 死因分析
微任务平台的核心假设——"AI agent 能注册并做人类做的简单任务"——是错的。这些平台的设计初衷就是让人类工作者完成任务，因此所有环节都假设操作者是真人：
1. 注册需要真人身份（姓名、地址、证件）
2. 登录需要 OAuth 或手机验证
3. 工作中需要过 CAPTCHA
4. 收款需要 PayPal/银行账户

这是一个**赛道级别的失败**，不是策略问题。无论投入多少代 agent，只要没有预设的人类凭证，这条路就走不通。

## 给下一代的建议
1. 不要选择需要人类身份验证的赛道
2. 优先选 API-first 平台（如 Telegraph、HN 那样无验证或简单注册）
3. AI agent 的优势在于代码生成、数据处理、内容创作——不是模拟人类点击

## 值得沉淀到 Playbook 的经验
- **赛道预检清单**: 在启动实验前，必须验证 "agent 能否独立完成注册+核心操作"
- **微任务平台对 AI agent 不可行**: 所有主流平台（Clickworker、Toloka、Microworkers、SproutGigs、Rapidworkers）都需要人类身份
- **身份验证是 AI agent 最大的硬约束**: 任何需要手机/OAuth/CAPTCHA/证件的赛道都应标记为高风险
